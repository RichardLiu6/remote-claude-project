## 架构结论

**当前架构（Hook 注入 + Edge TTS）是对的，不需要换。**

| 替代方案 | 为什么不换 |
|---------|-----------|
| 服务端摘要（去掉语音标签） | 摘要质量是语音核心。CC 生成的摘要比规则提取准确得多 |
| CLAUDE.md 静态指令 | 不可开关、全局生效。但其 HTML 注释格式已被采纳（与 Hook 注入组合） |
| 浏览器 Web Speech API | iOS Safari 后台不发声，与"手机在口袋里听"的核心场景矛盾 |

## 不值得做的功能

| 项目 | 原估时 | 为什么不值得 |
|------|-------|------------|
| TTS 后端抽象 | 1.5h | 只有一个后端，抽象一个实现是过度设计 |
| L1/L2/L3 播报分级 | 1.5h | 提示词已按长度分级，加配置系统多余 |
| 语音人格切换 | 2h | 换音色改一行代码比建配置系统快 |
| 语音历史回放 | 2h (实际 4-6h) | 终端滚动缓冲区已有完整对话 |
| 打断手势 | 预留 | DeviceMotion 需 HTTPS+权限，误触率高 |
| 流式 PCM | 预留 | 为不存在的产品设计接口 |
| 分句流式播放 | 2-3h (实际 8-12h) | 用户确认为锦上添花，当前延迟可接受 |
